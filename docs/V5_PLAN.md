# V5 Implementation Master Plan

> 基于 2026-05-12 §4 主要产品与功能需求要点逐条审计的结果，对未覆盖/部分覆盖的 13 项做体系化补齐。

**Goal:** 把当前 V4 完成后的实现，从 §4 (18 条要求) 的"1 完整 / 13 部分 / 4 缺失"提升到至少"13 完整 / 4 部分 / 1 暂缓"。

**Architecture:** 跨 manage + scan 两个仓库分 4 个阶段实施。manage 端补模版能力，scan 端补立项体验与 AI 归目深化。每阶段独立可交付、独立可验收，不阻塞既有 V1-V4 流程。

**Tech Stack:** manage (Nuxt 4 + better-sqlite3) + scan (Wails v2 + Go + Vue/Vuetify 3 + sqlx + Gin)

## 二、4 阶段拆分

### Phase 1 — Quick Wins（A1-A4，4 项）

**目标**：消除小颗粒度功能缺口，主要在 scan 端。

**详细计划**：`docs/superpowers/plans/2026-05-12-v5-phase-1-quick-wins.md`

- 任务 1-4：A1 TPL-PERSONAL-FILES 双环节 + 桥接区分过程/定稿
- 任务 5-7：A2 AI 归目驳回 + 调整端点 + UI
- 任务 8-11：A3 解除绑定 + 重新归类 + 原因登记
- 任务 12-13：A4 家族归档过程/定稿分流
- 任务 14：端到端验收

**预计工作量**：1.5-2 天 / 60-70 步骤

### Phase 2 — 立项体验提升（A1+B1+B2+B3，4 项）

**目标**：把"数据项目立项管理工具" §4.2 跟需求对齐。

**详细计划**：`docs/superpowers/plans/2026-05-12-v5-phase-2-project-wizard.md`（待 P1 完成后展开）

- B3 引入"数据项目样板" SMP-* 层（schema + manage CRUD + scan 选样板）
- B2 立项向导"常用排序 + 环节/规则开关 + 另存为个人模版"
- B1 项目工作事项认领（按角色筛 + 选定环节 + 自动 ensure 本地工作目录）
- "底账预览"补到立项确认页

**预计工作量**：3-4 天 / 80-100 步骤

### Phase 3 — AI 归目深化（B4+B5，2 项）

**目标**：把"项目版本文件 AI 归目工具" §4.3 提升到真正可用。

**详细计划**：`docs/superpowers/plans/2026-05-12-v5-phase-3-ai-classify-deepening.md`

- B4 AIClassifyView 按"个人关联项目（在进行中 + 最新优先）"分组重排
- B5 属性抽取扩展：
  - 文件元数据（mtime / size / mime / 创建人）
  - 路径上下文（最近活动目录、同目录批量打标）
  - 正文片段（仅对 .docx / .xlsx / .pdf 首页 200 字，本机 OCR 不接入第三方）
- 重排打分函数纳入新维度

**预计工作量**：2-3 天 / 50-60 步骤

### Phase 4 — 模版能力增强（C1-C4，4 项）

**目标**：把"数据业务模版定制与维护工具" §4.1 补到位。

**详细计划**：`docs/superpowers/plans/2026-05-12-v5-phase-4-template-capabilities.md`

- C1 模版级九宫格预设：在 template_file_rules 加 storage_baseline JSON 字段
- C2 模版授权层级化：完整 template_roles 表（角色规范）+ 立项时映射真名人员
- C3 评审环节：状态机 draft → review → active，加 template_reviews 表
- C4 三级派生：scope=industry/unit/department/person，发布范围控制

**预计工作量**：3-4 天 / 70-90 步骤

## 三、阶段依赖

```
P1 (Quick Wins, 含 A1 双环节)
     │
     ├──> P4 (模版能力增强，更动 template 结构)
     │         │
     │         └──> P2 (立项体验，依赖新 template 字段)
     │
     └──> P3 (AI 归目深化，独立)
```

**建议执行顺序**：P1 → P4 → P2 → P3。  
P3 可与 P4 并行。如人手紧张，按线性。

---

## 四、跨阶段约束

1. **不破坏既有数据**：所有 schema 改动走 migration，回滚兼容
2. **不动扫描文件**：CLAUDE.md 红线
3. **TDD**：每任务先写失败测试，再实现，最后绿
4. **频繁提交**：每个 task 一个 commit（或细分到 step 级别）
5. **测试前先 `npm rebuild better-sqlite3`**（manage 端）
6. **前端用 yarn**
