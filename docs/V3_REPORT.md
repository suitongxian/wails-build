## 5. 不在 V3 范围（明确边界）

### 5.1 §18 七条待确认问题 → V4

文档明列"以下问题不阻塞 MVP，但需要在后续版本明确"：

1. 项目编码规则是否企业自定义 / 系统统一生成
2. 三主体对接组织架构系统 vs 内部主体表
3. 文件实体存储优先用本地 / 对象存储 / 外部网盘
4. 归档包格式是否满足特定档案标准
5. 嵌入式文件属性打标支持哪些文件类型
6. 核心 / 重要数据目录上报的目标格式和接口
7. 是否需要与现有审批系统 / 合同系统 / OA 系统集成

V3 严格停在文档已定的范围内，**不对 §18 任何一条做技术押注**。

### 5.2 §12 五项暂不实现 → V5

1. 真实涉密系统对接
2. 文件内容嵌入式打标
3. 复杂 AI 自动归目
4. 跨域 / 跨网 / 跨地 / 跨境监管自动化
5. 第三方审批流深度集成

### 5.3 §11.1 部分类目接入留 V4

第 1 类（模版操作）和第 6 类（权限配置变更）的 audit_logs 实际写入
留 V4 — V3 阶段只把 Action 常量和表结构准备就绪，等 manage 侧
要做时按 V3-5 的范式接上即可。
