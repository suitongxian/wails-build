# 数据业务治理系统（终端）—— 数据业务服务 演示操作手册

> 适用范围：本手册只覆盖**终端（数据可信终端 / 数据业务治理系统）**里「数据业务服务」这一条业务线，供演示讲解人照着逐步操作。
> 阅读对象：第一次接触本系统、需要上台演示的同事。照着做即可，不需要懂代码。

---

## 一、一句话说清这套流程

> **把"一件数据业务"按模版拆成若干环节和文件任务，分给团队不同的人去做，每个人在终端里完成自己的文件、定稿归档，全程留痕。**

整个演示讲一条主线就够了：

```
项目立项  →  （后台审核通过）  →  负责人承接  →  组工作团队  →  关联模版  →  环节分工
        →  环节负责人再分工到具体文件任务  →  参与人做文件、交付定稿  →  在工作组里看到全队
```

---

## 二、演示前的准备

### 1. 角色与账号（建议准备 3 个账号）

本系统**按登录账号判断你现在是谁**，所以演示时需要**切换账号登录**（退出 → 换账号登录）来扮演不同角色。

| 演示账号（示例） | 扮演角色 | 在系统里的术语 | 主要在哪个菜单操作 |
| --- | --- | --- | --- |
| `zhang`（张组长） | 项目负责人 | **项目负责人 / 组长（lead）** | 项目立项、环节分工 |
| `li`（李环节） | 环节负责人 | **核心成员 / 环节责任人（core）** | 任务指派 |
| `wang`（王参与） | 文件任务参与人 | **项目参与人（participant）** | 工作受理 |

> 说明：演示时这 3 个账号要**事先在后台注册好**（带姓名、单位、部门）。登录后左上角标题显示「**数据业务治理系统**」。

### 2. 模版要先有（关键前置）

「关联模版」这一步要能选到模版，所以演示前请确认**模板库里至少有一套可用模版**（本地模版或在线模版均可）。

- 入口：左侧导航 **数据业务服务 → 模板库**
- 一套模版的结构是五层：**项目 ▸ 工作环节 ▸ 文件任务 ▸ 文件标识**
  - 演示时可以现成用一套，比如「环节：收稿 / 校对 / 归档」，每个环节下挂几个「文件任务」，文件任务下再挂「文件标识（要产出的定稿文件）」。

### 3. 找到入口

登录后看左侧导航栏 **「数据业务服务」** 分组，展开后有：

| 菜单 | 谁用 | 干什么 |
| --- | --- | --- |
| **项目立项** | 任何人（发起人） | 填立项书、定负责人，提交立项 |
| **环节分工** | 项目负责人 | 承接项目、组团队、关联模版、给各环节派负责人 |
| **任务指派** | 环节负责人 | 给本环节的每个文件任务派参与人 |
| **工作受理** | 文件任务参与人 | 做自己的文件任务：开始、编辑、完成、定稿 |
| **工作组** | 团队任何人 | 查看本项目工作组成员和角色 |
| **模板库** | 模版管理员 | 维护可用的数据业务模版（前置准备用） |

## 三、完整演示脚本（按这个顺序讲）

下面每一步都标注了：**【谁登录】**、**【在哪个菜单】**、**【点什么】**、**【会看到什么】**、**【讲解话术】**。

---

### 第 1 步：项目立项（发起一件数据业务）

- **【谁登录】** 张组长 `zhang`
- **【在哪个菜单】** 数据业务服务 → **项目立项**
- **【操作】**
  1. 点右上角 **「新建立项书」**。
  2. 弹出公文式立项书，填写：
     - **项目名称**（必填）：例「2026 年度客户档案数据治理」
     - **立项编号**：留空即可，发布后系统自动生成
     - **项目负责人**（必填）：下拉选 `张组长`（选中后**所属部门会自动带出**）
     - **敏感级别**（必填）：核心 / 重要 / 一般，演示选「重要」
     - **数据权属**（选填）：数据归谁所有，例「客户服务部」
     - **立项依据 / 项目简介**（选填）：随便填一两句
  3. 点 **「发布」**（也可「存草稿」先存着）。
- **【会看到什么】** 列表里出现这条立项，状态 **「处理中」**，并显示已推送到后台。
- **【讲解话术】**
  > "立项就是把一件要做的数据业务登记下来，最关键是**指定项目负责人**——指定的这一刻，团队其实就已经成立了，只是现在团队里只有负责人一个人。"

---

### 第 2 步：后台审核通过（必要的中转动作）

- **【说明】** 立项发布后会推送到**后台管理端**审核。这一步不在终端做，演示时可由后台同事提前点「通过」，或现场切到后台通过。
- **【回到终端】** 张组长在「项目立项」页点 **「刷新」**，系统会**同步后台审核结果**，这条立项状态变为 **「已立项」**。
- **【讲解话术】**
  > "审核通过后，这个项目才会下发给负责人去承接。"

> 💡 演示提示：如果只想演示终端、不想切后台，可以提前把这条立项审核好，演示当天直接从"已立项 / 待承接"开始讲。

---

### 第 3 步：承接项目（负责人接下任务，先看立项书）

- **【谁登录】** 张组长 `zhang`
- **【在哪个菜单】** 数据业务服务 → **环节分工**
- **【会看到什么】** 列表里这条项目状态 **「待承接」**，右侧操作列有 **「承接」** 按钮。
- **【操作】**
  1. 点 **「承接」**。
  2. 弹出 **立项书详情**（项目名称、编号、负责人、部门、敏感级别、数据权属、立项依据、简介等，**只读**）。
  3. 看完点 **「确认承接」**。
- **【会看到什么】** 状态变为 **「已承接」**，操作列出现 **「工作团队」** 和 **「关联模版」** 两个按钮。
- **【讲解话术】**
  > "点承接会**先展示立项书**，让负责人确认清楚再接。承接后他就正式成为项目负责人，可以继续往团队拉人、关联模版、分工。"

---

### 第 4 步：组建工作团队（往团队里拉人）

- **【谁登录】** 张组长 `zhang`
- **【在哪个菜单】** 环节分工 → 这条项目操作列点 **「工作团队」**
- **【会看到什么】**
  - 弹出「工作团队」窗口，**团队里已经有张组长本人**，角色标着 **「项目负责人」**，并且**带「必选」标记、不可移除**。
  - 这正是"立项即成队"的体现——不再有空团队、也没有"组建团队"的空状态。
- **【操作】**
  1. 在左侧人员列表里勾选要加入的人（例：李环节 `li`、王参与 `wang`）。
  2. 点 **「保存团队」**。
- **【会看到什么】** 提示"团队已保存（N 人）"。
- **【讲解话术】**
  > "团队不是从零组建，立项指定负责人时就已经有人了。负责人在这里继续把环节负责人、参与人拉进来。**谁在团队里，后面分工时才能被选到**——这是一道'人池'的闸门。"
- **【角色会自动标注】** 当某人后来被指派为环节负责人，他在团队里会显示「**核心成员**」；被指派为文件任务参与人，会显示「**项目参与人**」。

---

### 第 5 步：关联模版（给项目套上数据业务骨架）

- **【谁登录】** 张组长 `zhang`
- **【在哪个菜单】** 环节分工 → 操作列点 **「关联模版」**
- **【操作】**
  1. 弹窗上半部分再次展示**立项书摘要**（便于对照）。
  2. 下方下拉 **「关联模版」**：选一套模版（标「本地 · …」或「在线 · …」）。
  3. 点 **「确定」**。
- **【会看到什么】** 提示"已选定模版"，操作列的 **「关联模版」** 变成 **「分工」**。
- **【讲解话术】**
  > "模版就是这件数据业务的**分类骨架**——它规定了要走哪几个工作环节、每个环节要产出哪些文件。关联模版后，系统就知道这个项目该拆成哪些环节和任务了。"

---

### 第 6 步：环节分工（给每个工作环节指派负责人）

- **【谁登录】** 张组长 `zhang`
- **【在哪个菜单】** 环节分工 → 操作列点 **「分工」**
- **【操作】**
  1. 弹窗按模版列出全部**工作环节**（例：1. 收稿、2. 校对、3. 归档）。
  2. 每个环节右侧下拉选一名 **环节负责人**——**只能从刚才的工作团队里选**（如李环节）。
  3. 若发现团队里人不够，点 **「临时拉人入队」** 可当场补人（永久入队）。
  4. 全部环节都选好后点 **「提交」**。
- **【会看到什么】** 提示"分工成功，已下发环节任务"，项目状态变为 **「已分工」**，操作列变 **「已分工」**（可点查看进度）。
- **【讲解话术】**
  > "负责人把每个环节交给一名环节负责人。注意候选人**只来自工作团队**，保证分工对象都是事先纳入团队的人。提交后，这些环节任务就下发到对应环节负责人的终端了。"

---

### 第 7 步：环节负责人再分工（派到具体文件任务）

- **【谁登录】** 李环节 `li`（切换账号登录）
- **【在哪个菜单】** 数据业务服务 → **任务指派**
- **【会看到什么】** 列表里出现指派给他的工作环节（例「校对」），每行有 **「工作团队」** 和 **「分工」** 两个按钮。
- **【操作】**
  1. （可选）先点 **「工作团队」**：可以看到本环节团队里**已有李环节本人**，角色「**环节负责人**」、不可移除；可继续拉人、保存。
  2. 点 **「分工」**：弹窗列出本环节下的所有**文件任务**（例：录入、核对）。
  3. 每个文件任务右侧选一名 **工作参与人**——**只能从本环节团队里选**（如王参与）。人不够同样可「临时拉人入队」。
  4. 点 **「提交」**。
- **【会看到什么】** 提示"已提交分工"。
- **【讲解话术】**
  > "和项目负责人的流程一模一样：环节负责人被指派的那一刻，环节团队里就已经有他自己了，他再把本环节的每个**文件任务**派给具体参与人。这样就形成了'项目→环节→文件任务'三级分工。"

---

### 第 8 步：参与人做文件任务（核心演示亮点）

- **【谁登录】** 王参与 `wang`（切换账号登录）
- **【在哪个菜单】** 数据业务服务 → **工作受理**
- **【会看到什么】** 页面顶部是三个 **Tab：待办 / 进行中 / 已完成**（点哪个标签看哪一类，每个标签上有数量角标）。指派给他的文件任务出现在 **「待办」**。
- **【操作 8.1 开始工作】**
  1. 在「待办」里找到任务，点 **「开始工作」**。
  2. 系统会**按模版自动预建好该任务要产出的过程文档**（提示"已开始工作，按模版预建 N 个过程文档"），任务移到 **「进行中」**，并自动打开文件窗口。
- **【操作 8.2 编辑文档】**
  1. 在「进行中」里点 **「打开文档」**。
  2. 窗口分两块：
     - **工作依据**（上游环节交付的文件，点击用本机 Office/WPS 打开**查看**）
     - **过程文档**（本任务要写的文件，点击用本机 Office/WPS 打开**编辑**，本机保存即生效）
  3. 若本机没装 Office/WPS 打不开，窗口会浮出 **「在线编辑 / 在线查看」** 兜底按钮，可在网页里直接编辑保存。
- **【操作 8.3 完成并定稿】**
  1. 编辑完点 **「完成」**。
  2. 若本任务有**要产出的定稿（文件标识）**，会弹「提交定稿」窗口：为每个定稿，从你刚编辑过的过程文件里**各挑一个**作为最终定稿。
  3. 点 **「提交定稿并完成」**。任务移到 **「已完成」**。
  > 若该任务没有定稿要求，点「完成」会直接完成，不弹定稿窗。
- **【讲解话术】**
  > "参与人不用关心目录在哪、文件叫什么——系统按模版**自动建好过程文档**。他优先用本机 Office/WPS 编辑，打不开还能在线编辑兜底。完成前**谁编辑谁挑定稿**，把过程稿确定为最终成果，全程自动归档、留痕。"

---

### 第 9 步：工作组总览（收尾，展示团队全貌）

- **【谁登录】** 团队任意成员（张组长 / 李环节 / 王参与都行）
- **【在哪个菜单】** 数据业务服务 → **工作组**
- **【会看到什么】** 你参与的项目工作组，按角色分组展示：
  - **组长** = 项目负责人
  - **核心成员** = 各环节责任人
  - **参与人员** = 各文件任务参与人
- **【讲解话术】**
  > "最后回到工作组，能清楚看到这件数据业务的整支队伍和每个人的角色，分工一目了然。"

## 五、术语速查（讲解时统一口径）

| 界面叫法 | 含义 | 角色英文（备查） |
| --- | --- | --- |
| 项目负责人 / 组长 | 接下整个项目、组团队、分工的人 | lead |
| 核心成员 / 环节责任人 | 负责某个工作环节、再往下派任务的人 | core / stage_lead |
| 项目参与人 / 工作参与人 | 干具体文件任务的人 | participant |
| 工作环节 | 模版里的一个阶段（如收稿、校对） | stage |
| 文件任务 | 环节下要做的一件具体活 | task |
| 文件标识 / 定稿 | 文件任务要产出的最终成果文件 | file_rule / output |
| 工作团队 | 项目（或环节）的人池，分工只能从中选人 | team |

## 六、常见提问与应答（预案）

- **问：为什么没有"组建团队"按钮，直接是"工作团队"？**
  答：因为**立项指定负责人时团队就已经成立**了（里面至少有负责人本人）。所以不存在"空团队"，按钮统一叫「工作团队」，进去就能看到已有的人，继续拉人即可。

- **问：分工时下拉里怎么没有想选的人？**
  答：分工**只能从工作团队里选人**。请先在「工作团队」里把他拉进来，或在分工弹窗里点「临时拉人入队」当场补人。

- **问：参与人点「开始工作」后文件在哪？**
  答：系统按模版**自动预建好过程文档**，在「打开文档」窗口里就能看到，优先用本机 Office/WPS 编辑。

- **问：本机打不开文件怎么办？**
  答：窗口会浮出「在线编辑 / 在线查看」按钮，可在网页里直接编辑保存兜底。

```
